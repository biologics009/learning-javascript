# "use strict"

* in 2009 when ECMAScript 5 (ES5) appeared. 
* It added new features to the language and modified some of the existing ones. 
* Most such modifications are off by default to keep the old code working. 
* We need to explicitly enable them with a special directive: "use strict".

## Rules for this declerative
* "use strict" or 'use strict' is at the top of your scripts or file, otherwise strict mode may not be enabled.
* There’s no way to cancel use strict.
* Browser doesn’t use strict by default.
* Can be put it inside function wrapper if an old browser does not work with "use strict" at the top.
* We don’t need to add the "use strict" directive, if we use advance language structure e.g. “classes” and “modules”.
* classes and modules enable "use strict" automatically.
